Key Observations

Parallel Track 1

S1 (Grid) and S2 (Clock) are independent after S0. They can be built simultaneously. They converge at S3 where the game engine wires them together.

Parallel Track 2

S7 (Themes) only depends on S0 (the variable contract). Theme CSS can be developed in parallel with everything else, applied at any point.

Critical Path

S0 → S1 → S3 → S4 → S5 → S6 is the longest chain. S4 (AI Integration) is the most complex sprint on the critical path. Plan extra time there.

First Playable

End of S3 = first playable game (self-scored). End of S4 = AI-judged game. End of S5 = complete MVP with scoring.

04 — Risky Unknowns

What Could Go Wrong

Ranked by impact × likelihood. Each risk has a mitigation strategy and a "spike" (small experiment to validate before committing).

RiskLevelImpactMitigationSpike to Validate
AI judging inconsistency High Same answer gets different rulings across games. Player trust collapses. temperature: 0, aggressive caching (once judged = locked forever), appeal system with Sonnet. Run 50 identical validation requests for edge cases (The Beatles, New York, JFK). Measure variance. If >5% inconsistency, add few-shot examples to system prompt.
AI returns malformed JSON High Validation fails entirely. Player's round is wasted. Strict JSON parsing with try/catch. One retry with "respond with JSON only" reinforcement. Fallback to self-scoring. Send 20 validation batches to Haiku with the exact system prompt. Count JSON parse failures. If >5%, experiment with structured output / json_mode if available.
CORS / browser API access Med Direct browser fetch to Anthropic API blocked by CORS. Game can't validate at all. anthropic-dangerous-direct-browser-access: true header. Proven pattern from CPA Study Buddy. Single fetch call from a bare HTML file to api.anthropic.com/v1/messages. Confirm 200 response. Do this FIRST in Sprint 0 before building anything else.
ES modules without server Med import/export fails on file:// protocol. Player can't just double-click index.html. Require python3 -m http.server or VS Code Live Server. Document in README. Or: consider single-file build for distribution. Test: can 8 ES module files import each other via python3 -m http.server on macOS? Confirm Safari + Chrome compatibility.
Google Fonts loading latency Med Theme switch causes flash of unstyled text. Neon Arcade loads Orbitron and it's slow. font-display: swap in CSS. Preload the default theme's fonts. Lazy-load others only when selected. Show a brief loading indicator on theme switch. Load all 8 theme font sets and measure time-to-render on a slow 3G throttle. If any theme takes >2s, preload its primary font or provide a system-font fallback.
localStorage quota (5MB) Low At ~2KB/game + ~200B/cache entry, quota hit after ~2000 games + thousands of cached answers. LRU eviction for validation cache. History export before purge. Monitor storage size. No spike needed — this only becomes a problem at scale. Add a storage.js utility to check quota early.
Category/letter dead cells Low "Opera Composers starting with X" — impossible cell frustrates player. Curate categories to avoid narrow + rare letter combos. Selection algorithm can cross-check. Allow empty cells without penalty (just fewer points via n²). Generate 100 random grids with the proposed category pool. Have Claude evaluate: how many cells are genuinely impossible? Target: <5% dead cells per grid on Standard mode.
Haiku model deprecation Low Anthropic retires claude-haiku-4-5 model ID. Model ID is a single constant in claude-api.js. One-line change. Settings screen could allow model override. No spike needed — just keep the model ID in one place.

Recommended Spike Order

Before writing any game code, run these 3 experiments (< 1 hour total):

Spike 1: API Access from Browser (10 min)

Single HTML file. One fetch to Anthropic API with the dangerous-direct-browser-access header. Confirm 200 response in both Safari and Chrome. If this fails, the entire architecture changes (would need a proxy server). This is the highest-risk validation.

Spike 2: Validation Consistency (20 min)

Send the exact system prompt + 10 edge cases to Haiku 20 times each (200 calls). Parse responses. Measure: (a) JSON parse success rate, (b) ruling consistency for identical inputs. Target: >98% valid JSON, >95% consistent rulings. If inconsistent, add few-shot examples or switch to Sonnet.

Spike 3: Category Viability (15 min)

Generate 20 random grids with the proposed category pool. Ask Claude to evaluate each cell: "Is there a valid answer for [category] starting with [letter]?" Count impossible cells. Target: <5% on Standard, <15% on Expert. If too many dead cells, expand the category pool or add letter-compatibility metadata.

Validation Checklist

Is the core mechanic proven? Yes — 150+ years as a parlor game, 60 years as a commercial product, millions of Scattergories copies sold. The mechanic works. We're digitizing, not inventing.

Is the tech stack appropriate? Yes — vanilla web for a grid + timer + API calls is right-sized. No framework overhead. Matches the developer's prior shipped projects.

Is the AI integration feasible? Likely yes — CPA Study Buddy proves the browser-to-API pattern. Haiku handles classification well. Needs spike validation for consistency.

Is the scope achievable? Yes — the MVP (S0–S5) is 6 sprints of focused work. Each sprint has clear inputs and outputs. No ambiguous requirements. Theme work (S7) is independent and can be deferred.

Open question: ES modules on file:// — Need to confirm the python server workflow is acceptable, or consider a single-file build alternative for easy distribution. Low risk, but worth deciding upfront.

Open question: Default theme. Which of the 8 themes ships as default? This affects first impressions significantly. Recommend: Retro Bookshelf (nostalgic, warm, matches the game's heritage) or Newspaper (intellectual, crossword-adjacent, familiar to target audience).

06 — Critical Path Pseudocode

The 3 Hardest Functions

If these work, everything else is straightforward.

1. The n² Score Calculator

function calculateScore(grid, validationResults) {
  // grid[row][col] = { answer }
  // validationResults[row][col] = { valid: bool }

  const rowTotals = [0, 0, 0, 0, 0]    // General Score (breadth)
  const colTotals = [0, 0, 0, 0, 0]    // Special Score (depth)

  for (let r = 0; r < 5; r++) {
    for (let c = 0; c < 5; c++) {
      if (validationResults[r][c].valid) {
        rowTotals[r]++
        colTotals[c]++
      }
    }
  }

  const general = rowTotals.reduce((sum, n) => sum + n * n, 0)  // Σ rows²
  const special = colTotals.reduce((sum, n) => sum + n * n, 0)  // Σ cols²

  return {
    rowTotals,              // [5, 4, 5, 3, 4]
    colTotals,              // [5, 4, 4, 4, 4]
    rowScores: rowTotals.map(n => n * n),  // [25, 16, 25, 9, 16]
    colScores: colTotals.map(n => n * n),  // [25, 16, 16, 16, 16]
    general,                // 91
    special,                // 89
    total: general + special,  // 180
    max: 250,
    validCount: rowTotals.reduce((a, b) => a + b, 0),
    totalCells: 25,
  }
}

2. The Validation Prompt Builder

function buildValidationRequest(grid, categories, letters) {
  const systemPrompt = `You are the judge for "Facts in Five."
Rules:
1. Answer must belong to the category.
2. Answer must start with the specified letter.
   - People: SURNAME is the keyword (e.g., "Einstein" for E).
   - Titles: ignore "The", "A", "An" (e.g., The Birds → B).
   - Places: first word (e.g., New York → N).
3. Must be real and verifiable — no fictional entries.
4. Common abbreviations/nicknames OK if widely recognized.
5. Minor spelling errors OK if clearly recognizable.

Respond with a JSON array only. No markdown fences.
Each element: {"id":"rXcY","valid":boolean,"explanation":"...","canonical":"..."}`

  const answers = []
  for (let r = 0; r < 5; r++) {
    for (let c = 0; c < 5; c++) {
      const answer = grid[r][c].answer.trim()
      if (answer) {
        answers.push({
          id: `r${r}c${c}`,
          category: categories[r].name,
          letter: letters[c],
          answer: answer,
        })
      }
    }
  }

  // Check cache first — only send uncached answers
  const { cached, uncached } = partitionByCache(answers)

  if (uncached.length === 0) return { allCached: true, results: cached }

  return {
    allCached: false,
    systemPrompt,
    userMessage: `Validate:\n${JSON.stringify(uncached)}`,
    cachedResults: cached,
  }
}

3. The Wall-Clock Timer

function createTimer(durationSeconds, onTick, onExpiry) {
  const endTime = Date.now() + (durationSeconds * 1000)
  let running = true
  let rafId = null

  function tick() {
    if (!running) return
    const remaining = Math.max(0, endTime - Date.now())
    const seconds = Math.ceil(remaining / 1000)

    onTick({
      remaining,             // ms remaining (for progress bar precision)
      seconds,               // whole seconds (for digit display)
      fraction: remaining / (durationSeconds * 1000),  // 0..1 for bar
      isWarning: seconds <= 60 && seconds > 15,
      isCritical: seconds <= 15,
    })

    if (remaining <= 0) {
      running = false
      onExpiry()
      return
    }

    rafId = requestAnimationFrame(tick)
  }

  // Handle tab visibility
  function onVisibilityChange() {
    if (document.hidden) {
      // Tab hidden — stop RAF (browser throttles anyway)
      if (rafId) cancelAnimationFrame(rafId)
    } else {
      // Tab visible — restart ticking (wall clock still correct)
      tick()
    }
  }

  document.addEventListener('visibilitychange', onVisibilityChange)
  tick()  // Start

  return {
    stop() {
      running = false
      if (rafId) cancelAnimationFrame(rafId)
      document.removeEventListener('visibilitychange', onVisibilityChange)
    }
  }
}
