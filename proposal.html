01 — Product Proposal

Vision

F5 is the definitive digital version of the category-letter grid game — a genre that has existed for 150 years but has never had a proper modern home. By combining the original game's elegant n² scoring system with AI-powered answer validation, F5 eliminates the single biggest friction point of the physical game (judging disputes) while preserving everything that made it a favorite of JFK's Camelot circle.

Target Audience

Primary

Adults 30–65 who grew up with word games, crosswords, and trivia. The "Wordle demographic" — people who want a daily intellectual ritual that respects their intelligence.

Secondary

Families and friend groups looking for a shared game night experience. The Class/Category difficulty hierarchy lets kids and adults compete on the same grid.

Tertiary

Educators and cognitive health advocates. The game exercises lexical retrieval, category fluency, and processing speed — brain training that doesn't feel like homework.

Core Value Proposition

The 5×5 grid is the differentiator. Every competitor (Scattergories, Stop, Fight List) simplifies to "multiple categories, one letter." F5 is the only digital game doing the full 5×5 matrix — where every cell is a unique intersection of category and letter. The AI judge makes this playable solo for the first time in the game's 60-year history.

Feature Roadmap

MVP

Phase 1 — Solo Play

5×5 grid, 5-minute timer, curated category pool, random letter selection, Claude API validation, n² scoring, personal best tracking, local storage persistence. A complete, shippable game.

DEPTH

Phase 2 — Mastery

Daily Challenge mode (same grid for everyone, like Wordle). Difficulty tiers (Casual/Standard/Expert). Stats dashboard. Guggenheim variant (use a real word for column letters). Streak tracking. Appeal system for disputed rulings.

SOCIAL

Phase 3 — Multiplayer

Async play (share a grid, compare scores). Real-time head-to-head. Leaderboards. Server-side validation via Vercel serverless functions.

Key Differentiators vs. Competitors

  • The full 5×5 matrix — not a simplified single-letter variant
  • AI judge — eliminates arguments, enables solo play, handles edge cases with explanations
  • The n² scoring system — mathematically elegant, rewards depth AND breadth
  • Class → Category hierarchy — built-in difficulty scaling without separate "easy mode"
  • No predatory monetization — no energy systems, no loot boxes, no ads-for-hints
02 — Game Design

Core Loop

One round of F5 follows a clean five-step arc:

1. Setup

5 categories are selected (from the curated pool, respecting difficulty and variety rules). 5 letters are drawn randomly. The 5×5 grid is presented.

2. Play

The 5-minute timer starts. Player fills cells freely in any order — Tab moves across letters, Enter moves down categories. Every cell is a unique challenge: the intersection of its row's category and column's letter.

3. Validate

Timer expires. Grid locks. All answers are sent to Claude in a single API call. The AI validates each answer against its category and letter, returning valid/invalid + a one-line explanation.

4. Reveal

Results are animated cell-by-cell (drama!). Valid answers glow green. Invalid answers show the explanation. Empty cells stay dim.

5. Score

The n² scoring formula calculates Special Score (depth per category) and General Score (breadth per letter). Total is displayed with breakdown. Game is saved to history.

Letter Selection

5 unique letters drawn from the alphabet. Default mode excludes Q, X, and Z (the "fun-killers"). Expert mode includes them.

AI Judging Rules

Based on tournament rules refined over 30+ years of competitive play:

Keyword Edge Cases

"Grant, Ulysses S." for letter G — U.S. Presidents VALID
People: surname is the keyword. "G" for Grant.
"The Birds" for letter B — Movies VALID
Titles: leading articles (The, A, An) are ignored.
"LeBron James" for letter J — Athletes VALID
People: surname "James" is the keyword.
"New York" for letter N — Cities VALID
Places: first word of the name is the keyword.
"Beyonce" for letter B — Musicians VALID
Mononymous figures: the single name is the keyword.
"JFK" for letter K — U.S. Presidents VALID
Widely recognized abbreviation. Surname "Kennedy" starts with K.
"A Beautiful Mind" for letter A — Movies INVALID
Leading article ignored → keyword is "Beautiful" → valid for B, not A.
"pH" for letter P — Scientific Terms VALID
First character is P regardless of case conventions.
"foie gras" for letter F — Foods VALID
Foreign terms in common English usage are permitted.
"Narnia" for letter N — Countries INVALID
Must be a real, verifiable thing. Fictional entries are not accepted.

Difficulty Modes

Casual

Broad categories only (Animals, Colors, Foods). No hard letters. 7-minute timer. Great for kids and new players.

Standard

Mix of broad and specific categories. 5-minute timer. The classic experience.

Expert

Narrow categories (Jazz Musicians, Nobel Laureates). All 26 letters possible including Q, X, Z. 5-minute timer.

Daily Challenge

Same grid for every player, every day. Standard difficulty. Global comparison. One attempt only.

Guggenheim Variant

Instead of 5 random letters, the columns use the letters of a real 5-letter word (e.g., BRAIN, MATCH, QUEST). No repeated letters. The word itself becomes part of the puzzle's identity — "How'd you do on FLAME today?"

03 — The n² Scoring System

The Math of Incentives

Completing a row

Going from 4/5 to 5/5 in a row gains +9 points (16 → 25). That one answer is worth more than 3 scattered answers in empty rows.

Diminishing returns of scattering

5 answers across 5 different rows = 5 × 1² = 5 points. Those same 5 answers completing one row + one column = 25 + column improvement. Depth beats breadth.

Theoretical maximum

Perfect grid: 5 × 25 + 5 × 25 = 250 points. A score above 100 is strong. Above 200 is exceptional.

05 — Psychology & Engagement

Why This Game Works on Your Brain

Flow State

The 5×5 grid + 5-minute timer creates optimal flow. The challenge scales naturally with the player's knowledge — easy categories feel fast, hard ones feel urgent. The clock creates "productive anxiety" that prevents both boredom and overwhelm. Csikszentmihalyi's flow channel, perfectly calibrated.

Tip-of-the-Tongue Effect

The psychological phenomenon where you know you know something but can't retrieve it. In daily life, this is frustrating. In a game context, it's delicious — the near-miss creates tension that resolves either in a "got it!" dopamine hit or a "how did I not think of that?" laugh. Both are pleasurable.

Knowledge Identity

"I knew that!" is one of gaming's most powerful emotions. F5 lets players discover what they know — and feel smart about obscure knowledge paying off. The game is a mirror of your internal encyclopedia, and people love looking in that mirror.

Retrieval Interference

The dual constraint (category AND letter simultaneously) creates high cognitive load. Your brain has to inhibit irrelevant words while searching long-term memory for specific matches. This is why it feels like work in the best way — you're exercising a real cognitive skill.

Replayability

The combinatorial explosion guarantees freshness: with 60 categories and 23 letters, the number of possible grids is astronomical. But more importantly, the same grid played by different people produces wildly different answers. "World Capitals starting with B" — you wrote Brasilia, they wrote Bangkok, someone else wrote Bern. This divergence is inherently fascinating and drives social sharing.

Habit Formation (Daily Challenge)

The Daily Challenge mode leverages the same psychology as Wordle: one attempt, same puzzle for everyone, shareable results. The constraint of "one shot" makes each attempt feel meaningful. The universality enables comparison without competition. Streaks create commitment devices. The "grid share" (showing your pattern of hits/misses without spoiling answers) is a natural social object.

Danger Zones & Mitigations

Feeling stupid. The #1 reason people quit trivia games. Mitigate by always including 1–2 broad categories per round (even in Expert mode). Nobody should stare at a completely blank grid.

Unfair AI judging. If the AI rejects an answer the player believes is valid, trust collapses. Mitigate with the Appeal system (re-evaluate with a more powerful model) and always show the AI's reasoning.

Impossible combinations. "Opera Composers starting with X" is a dead cell. Mitigate by testing category/letter compatibility in the selection algorithm, and by offering partial-credit variants in Casual mode.

08 — Implementation Plan

Architecture & Tech Stack

Phase 1: Vanilla HTML + CSS + JS

No React. No bundler. No Node server. Open index.html in a browser and play. This matches the proven pattern from CPA Study Buddy — modular ES modules, direct Anthropic API calls from the browser using anthropic-dangerous-direct-browser-access: true, API key stored in localStorage via a setup screen.

Why This Stack

  • Zero friction. No npm install, no dev server, no build.
  • Proven pattern. Shipped multiple projects this way.
  • Sufficient. A 5×5 grid, a timer, and API calls. React would be overhead with no payoff.

Future Path

  • Phase 2: Add Supabase (JS SDK via CDN) for persistent stats and leaderboards.
  • Phase 3: Vercel serverless functions. Server holds API key for multiplayer. Frontend stays static.

AI Integration

Model Choice

claude-haiku-4-5 for validation. It's a classification task, not creative writing. 25x cheaper than Sonnet, fast enough for batch validation. Reserve Sonnet for appeals of disputed rulings.

Validation Strategy

Batch at round end, not real-time. The timer IS the pressure. Real-time validation removes it. One API call with all 25 answers after the clock expires. Response time via Haiku: ~1–2 seconds.

Prompt Structure

System: You are the judge for "Facts in Five."
Rules: surname for people, ignore articles for titles,
must be real/verifiable, English or common loanwords.
Respond with valid JSON only.

User: Validate these answers:
[
  {"id":"r0c0","category":"World Capitals","letter":"B","answer":"Budapest"},
  {"id":"r0c1","category":"World Capitals","letter":"M","answer":"Manila"},
  ...
]

Response: [
  {"id":"r0c0","valid":true,"explanation":"Budapest is the capital of Hungary.","canonical":"Budapest"},
  ...
]

Caching

Cache key: category|letter|answer (lowercase/trimmed). Stored in localStorage. Facts don't change — no TTL needed. After ~20 games, ~40% cache hit rate. Common answers (Paris, London, Beethoven) are validated once, forever.

Key Technical Decisions

Timer Precision

Store wall-clock end time, compute remaining on each tick. setInterval drifts; Date.now() doesn't. Handle visibilitychange for tab-switch.

AI Consistency

temperature: 0 for deterministic classification. Cache aggressively — once validated, result is locked forever. Appeal system (Sonnet) for the 2% of wrong calls.

Grid Navigation

Tab = next letter (right). Enter = next category (down). Shift+Tab = back. Click any cell freely. Spreadsheet-like speed.

Offline

Grid, timer, input: fully offline (just HTML/CSS/JS). Validation needs API — fallback to self-scoring with checkboxes. Categories bundled in code, not fetched.
